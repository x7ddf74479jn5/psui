# Introduction

PSUI は、React と Tailwind のために設計された UI ライブラリです。以下のような問題に対処することを目的としています：

- MUI、Chakra、Bootstrap のようなコンポーネントライブラリは、デフォルトのまま利用する分には素敵ですが、カスタマイズが難しい場合があります。
- Vanilla Tailwind や Flowbite のようなライブラリでは、テーマ設定が面倒になることがあります。すべてのコンポーネントにライトモードの色とダークモードの色を定義する必要があるのが嫌です。
- Vanilla Tailwind は、異なるスタイルを持つプロジェクト間での再利用性が低いです。
- Daisy UI のような CSS のみのライブラリは、アクセシビリティの問題があります。
- Radix や Headless UI のようなヘッドレスライブラリは、"ヘッドレス"です。要は、スタイルなしということなのですが、その分、スタイル付けに多くの時間を費やすことになります。

とはいえ、これらのライブラリにも好きなところはたくさんあります。Tailwind のスピードとカスタマイズ性、Daisy UI のテーマ性、Radix のアクセシビリティ、そして MUI のすぐに使えるデザイン。PSUI は、私がこれらのライブラリで気に入っている部分をすべて組み合わせ、問題点を最小限に抑える試みです。

Radix UI & Tailwind CSS の組み合わせの UI コンポーネントライブラリは<a href="https://twitter.com/shadcn">Shadcn</a>の<a href="https://ui.shadcn.com/">shadcn/ui</a>が有名です。出発点として shadcn/ui のコンポーネントの多くを使用しました。カラーシステムのない同様のライブラリをお探しなら、ぜひチェックしてみてください！

## Key Features

### Themeable & Reusable

PSUI は Tailwind を拡張し、コンポーネントがあらかじめスタイル付けされたセマンティックカラーシステムを追加しています。これにより、アプリ間でテーマの切り替えやコンポーネントの再利用が容易になります。詳しくはこちら：[カラーシステム](?path=/docs/concepts-colors--docs)。

### Accessibility

PSUI は、アコーディオンやセレクトなどのインタラクティブなコンポーネントに Radix Primitives を使用しています。詳しくはこちら：[アクセシビリティ](?path=/docs/concepts-accessibility--docs)。

### Customizable

コンポーネントには、インストール後即時に使える見栄えのする賢明なデフォルトスタイルが付属していますが、Tailwind ユーティリティクラスで完全にカスタマイズすることができます。PSUI では、[tailwind-merge](https://www.npmjs.com/package/tailwind-merge)を使用して、この処理を行います。詳しくはこちら：[カスタマイズ](?path=/docs/concepts-customization--docs)。
